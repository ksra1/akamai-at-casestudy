# SR. TPM Panel Interview Scenario

## Scenario Overview
- **Company**: AT Retailers (global e-commerce)
- **Geography**: North America, LATAM, Europe
- **Current State**: Monolithic architecture on on-prem + AWS
- **Transformation**: Moving to microservices/API-driven
- **Recent Activity**: Multiple acquisitions needing migration to Akamai

## Key Challenges Identified

### A) Delivery Issues
1. **30 days to onboard 5,000 hostnames** ✅ Challenge 1
2. **New teams not trained on Akamai**
3. **Handle 5x traffic during peak events**
4. **Slow image loading**
5. **Customer wants production-first (no testing)**
6. **Managing resources across multiple geographies** (Europe, US, Asia)

### B) Bot Activity & Automated Threats
1. DDoS attacks on origin
2. Legacy/outdated WAF rule sets
3. Credential stuffing attempts
4. Inventory scraping by competitive bots
5. Carding attempts on checkout APIs
6. Sophisticated bot evasion (IP rotation, spoofed clients)

## Interview Deliverables
- 30-minute structured presentation
- High-level solution architecture using Akamai products
- Project management style roadmap & execution plan
- Competitive advantages vs other vendors
- One Akamai best practice per product

## Required Presentation Sections
A) **Delivery & Project Management Strategy** - phases, coordination, change management, risk management
B) **High-Level Security Architecture** - onboarding, traffic/load solutions, WAF/Bot/API security
C) **Security Problem Analysis** - risks, business impact, how issues interrelate
